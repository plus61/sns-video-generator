# 📊 シンプリシティ・スコア実例

## TDDセッション with Worker2

### 概要
- 機能: 動画サムネイル生成
- 時間: 5分（Worker2は37秒で完了！）
- 結果: 完全動作

### スコア計算

```
シンプリシティ・スコア = (1000 - 総行数) × (1 / 実装時間[分]) × 機能完成度

実例：
- 総行数: 8行（テスト3行 + 実装5行）
- 実装時間: 5分
- 機能完成度: 100%

スコア = (1000 - 8) × (1 / 5) × 1.0
      = 992 × 0.2 × 1.0
      = 198.4 ポイント
```

### 比較

**従来のアプローチ（推定）：**
- 総行数: 150行（テスト50行 + 実装100行）
- 実装時間: 120分
- 機能完成度: 100%

スコア = (1000 - 150) × (1 / 120) × 1.0 = 7.08 ポイント

### 改善率
**198.4 ÷ 7.08 = 28倍の改善！**

## 考察

このスコアが示すのは、シンプルさが生産性に与える劇的な影響です。
8行のコードは、150行のコードより28倍「良い」のです。
